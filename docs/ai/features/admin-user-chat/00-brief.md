# Admin → User Chat Entry — Brief

## Цель
Дать администратору возможность начать диалог с пользователем из раздела «Пользователи» через кнопку «Отправить сообщение», используя уже реализованный `admin-chat`.

После отправки сообщения:
- сообщение сохраняется в существующем чате,
- у пользователя появляется индикация непрочитанного сообщения.

---

## Где

- Админ-панель  
  Раздел: Пользователи  
  Страница: список пользователей / карточка пользователя  

- Используется существующий модуль `support-chat` (без переписывания логики)

---

## Функционал

### 1. Кнопка «Отправить сообщение»

- Отображается в строке пользователя (или в карточке пользователя).
- Доступна только пользователям с правами администратора.
- При клике:
  - открывается существующий `admin-chat`,
  - чат открывается с конкретным пользователем,
  - если диалог уже существует → открывается существующий,
  - если не существует → создаётся новый диалог.

---

### 2. Поведение чата

- Используется текущая реализация `admin-chat`:
- Никаких изменений в логике сообщений не вносится.

---

## Архитектурные требования

- Не дублировать логику чата.
- Не создавать новый тип диалога.
- Использовать текущую модель диалогов (admin ↔ user).

---

## Авторизация

- Только пользователи с правом `canManageSupportChats` могут инициировать чат.
- Пользователь видит только свои диалоги.
- Исключить утечки чужих диалогов (IDOR).
