# Feature Brief: Support Chat — User Avatar + Full Name

## Goal
Добавить отображение аватарки и ФИО пользователя в существующий функционал support-chat:
- В списке диалогов (окно диалогов) показывать аватар и ФИО.
- В самом чате в шапке (header) показывать аватар и ФИО рядом с кнопкой "назад".

## Scope
- Только UI-изменения для отображения аватарки и ФИО.
- Без изменения бизнес-логики чата, SSE, вложений, read/unread, прав доступа.
- Без изменения API контрактов, если нужные данные уже доступны в текущем состоянии/DTO.

## UI Requirements

### 1) Dialogs List (окно диалогов)
- Для каждого диалога отображать:
  - слева: аватар пользователя
  - справа от аватара: ФИО пользователя
- Аватар должен рендериться через компонент `ProfileAvatar`.

### 2) Chat Header (шапка чата)
- В верхней части чата:
  - слева кнопка "назад" (существующая)
  - справа от кнопки "назад": аватар пользователя + ФИО пользователя
- Аватар должен рендериться через компонент `ProfileAvatar`.

## Component Constraint
- Обязательно использовать существующий компонент `ProfileAvatar`.
- Не создавать альтернативные компоненты аватарок и не дублировать логику отображения.

## Data Requirements
- Источник данных для `avatar` и `fullName`:
  - использовать то, что уже есть в текущих данных диалога/чата (initial state, query, DTO, store)
  - если в одном месте данные есть, а в другом нет, прокинуть их на UI уровень минимальным способом (без рефакторинга архитектуры)

## Definition of Done
- В списке диалогов отображаются аватар + ФИО для каждого диалога.
- В шапке чата отображаются аватар + ФИО выбранного диалога.
- Везде используется `ProfileAvatar`.
- UI выглядит корректно на mobile и desktop (без сдвигов и переполнений, текст ФИО не ломает верстку).

## Non-goals
- Не добавлять новые поля в БД и не менять модели, если это не строго необходимо.
- Не менять стили приложения глобально.
- Не добавлять новый функционал (редактирование, удаление, бейджи, фильтры и т.д.).

## Notes
- Если ФИО отсутствует, отображать безопасный fallback (например, "Пользователь") и не ломать верстку.
- Если аватар отсутствует, `ProfileAvatar` должен корректно отрисовать fallback (как предусмотрено компонентом).
